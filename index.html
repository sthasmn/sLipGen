sLipGen

Automated Active Speaker Detection 自動話者検出と
& Temporal Video Segmentation 動画の時間的セグメンテーション

A system for creating large-scale, Japanese lip-reading datasets from in-the-wild videos like TV programs. テレビ番組などの自然な映像から、大規模な日本語の読唇データセットを作成するためのシステム。

How It Works: The Full Pipeline 仕組み：完全なパイプライン

1. Input: In-the-Wild Video 1. 入力：自然な環境のビデオ

The process starts with any standard video file, typically Japanese TV programs. プロセスは、主に日本のテレビ番組など、あらゆる標準的なビデオファイルから始まります。

2. Pre-processing 2. 前処理

The video is split into two streams: individual image frames and a standardized audio track. ビデオは個々の画像フレームと標準化された音声トラックの2つのストリームに分割されます。

Frames Audio

3. Vision Analysis 3. ビジョン分析

Frames are analyzed for scene cuts, faces are detected with bounding boxes, and then tracked across frames. フレームはシーンカットのために分析され、顔がバウンディングボックスで検出され、フレーム間で追跡されます。

4. Speaker Scoring (ASD) 4. 話者スコアリング (ASD)

The ASD model synchronizes face tracks with the audio, scoring each face for how likely they are to be the speaker. ASDモデルが顔の軌跡と音声を同期させ、各顔が話者である可能性をスコアリングします。

5. Transcription 5. 書き起こし

OpenAI's Whisper transcribes the entire audio track, generating text with word-level timestamps. OpenAIのWhisperが音声トラック全体を書き起こし、単語レベルのタイムスタンプ付きテキストを生成します。

6. Final Assembly 6. 最終組立

High-scoring speakers are matched with transcribed sentences to generate final, labeled video clips and text files. 高スコアの話者と書き起こされた文が照合され、最終的なラベル付きビデオクリップとテキストファイルが生成されます。

7. Output: Lipreading Dataset 7. 出力：読唇データセット

The result is a structured folder of video clips, text files, and a `manifest.json`, ready for training ML models. 結果は、MLモデルのトレーニングに使用できる、ビデオクリップ、テキストファイル、`manifest.json`で構成された構造化フォルダです。

Setup & Usage Guide セットアップと使用ガイド

Option A: Using Docker (Recommended) オプションA: Dockerの使用（推奨）

1. Clone the Repository 1. リポジトリのクローン

git clone https://github.com/sthasmn/sLipGen.git
cd sLipGen

2. Build the Docker Image 2. Dockerイメージのビルド

docker build -t slipgen .

3. Run the Pipeline 3. パイプラインの実行

Choose one of the two modes below. 以下の2つのモードから1つを選択してください。

Mode 1: Direct Processing モード1: 直接処理
# Replace /path/to/your/videos with your actual path docker run --gpus all --rm -it \ -v /path/to/your/videos:/videos \ slipgen python main.py /videos/
Mode 2: Persistent Container (for SSH) モード2: 永続コンテナ（SSH用）
# Start the container docker run --gpus all -d --name slipgen-dev \ -v /path/to/your/videos:/videos \ -p 127.0.0.1:2222:22 \ slipgen # Connect via SSH (User: lipgen, Pass: lipgen) ssh lipgen@localhost -p 2222
